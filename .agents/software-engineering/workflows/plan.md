## /plan — Software Engineering Planning (RPM Protocol)

1. **RESULT**: Kết quả kỹ thuật cụ thể (API mới, tính năng X, fix bug Y).
2. **PURPOSE**: Tại sao cần làm? Business impact là gì?
3. **MAP**: Lộ trình thực hiện:
   - Phân tích codebase hiện tại (đọc file liên quan trước)
   - Thiết kế kiến trúc + Design Pattern phù hợp
   - Xác định files cần tạo mới / sửa / xóa
   - Viết code theo SOLID + DRY
   - Test và verify
4. Xác nhận với Sếp trước khi execute.
